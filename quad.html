ReconQuad

Overview

For my winter project, I designed and built a self-reconfigurable quadruped that can transform between walking and rolling mode. I also integrated ROS2 and YOLO for a computer vision task.

Project Sequence

  • The robot starts turning. At the same time, it is scanning for bowling pins. There is a 7s pause between turns.
  • If the robot detects at least 2 pins for at least 3 seconds continuously, transform to rolling.
  • Once it is in rolling mode, it uses the IMU data to get tilt angles and roll forward.

Nodes

The bowling task is divided into several subsystems:

Control (Action Server and Client Nodes)

These nodes updates the robot states (TURNING, STOPPING, TRANSFORM TO ROLL, ROLLING, etc.) based on the number of bowling pins and publishe the corresponding robot configuration for the Motor Control Node to set the DXL motors.

Computer Vision (Yolo Node)

This node processes data from the camera and count the number of bowling pins to be published to a topic.

Dynamixel Motor Control (Quad Motor Control Node)

This node controls the motors depending on the robot state and process IMU data to get tilt angles during ROLLING mode.
